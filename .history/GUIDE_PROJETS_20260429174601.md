## 🚀 Étapes pour ajouter vos projets

### Étape 1 : Préparer les images

Pour **chaque projet** (3 et 4), créez un dossier dans `images/` :

```
images/
├── projet3/
│   ├── principal.png (ou screenshot principal)
│   ├── img1.png
│   ├── img2.png
│   └── ...
│
└── projet4/
    ├── principal.png
    ├── img1.png
    ├── img2.png
    └── ...
```

**💡 Conseil** : 
- **Taille principale** : 800×400px minimum pour une bonne qualité
- **Format** : PNG ou JPG
- **Nombre** : 2-4 images par projet

### Étape 2 : Modifier le HTML

Ouvrez `Index.html` et trouvez la section `<!-- PROJET 3 -->` (vers ligne 450)

#### Pour le PROJET 3 :

Remplacez cette ligne :
```html
<img src="images/site_313/darkmode.png" class="w-full h-56 object-cover" alt="Projet 313">
```

Par :
```html
<img src="images/projet3/principal.png" class="w-full h-56 object-cover" alt="Mon Projet 3">
```

Remplacez le titre :
```html
<h3 class="text-xl font-bold mb-2 dark:text-white">Projet / Démonstration 3</h3>
```

Par votre titre (exemple) :
```html
<h3 class="text-xl font-bold mb-2 dark:text-white">API REST en Python</h3>
```

Remplacez la description :
```html
<p class="text-slate-600 dark:text-gray-400 mb-4 flex-grow">
  Ajoutez une image de votre projet...
</p>
```

Par votre description :
```html
<p class="text-slate-600 dark:text-gray-400 mb-4 flex-grow">
  Développement d'une API REST avec FastAPI et PostgreSQL.
```


## 🎨 Conseils pour les descriptions

Gardez vos descriptions **claires et percutantes** pour une RH :

✅ **BON** : "Développement d'une API REST avec FastAPI et PostgreSQL. Authentification JWT, documentation Swagger, tests unitaires."

❌ **MAUVAIS** : "J'ai fait un truc avec Python"

### Template de description :
```
[Domaine] - [Technos utilisées]. [Brève description du résultat].
```

Exemples :
- "Scripting Python - Automatisation d'analyses de logs avec regex et pandas"
- "Pentest - Configuration de lab DVWA pour exercices de sécurité"
- "Développement - Dashboard en React et TypeScript pour monitoring réseau"
